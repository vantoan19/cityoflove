# Chapter 5+6 — "The Part Where We Went Too Fast"
## Design Spec

**Date:** 2026-04-24
**Status:** Approved

---

## Overview

Chapters 5 and 6 are combined into a single scroll page with two distinct movements. Together they tell one story: *we went too fast → things broke a little → I get it now.*

- **Movement 1 (M1) — The Speedrun:** Self-aware, slightly chaotic, fast. The text IS the spectacle. Ends with a memory flash montage.
- **Movement 2 (M2) — The Glitch:** Tender, empathetic, slow. Quiet acknowledgment. The world exhales.

**Tech:** Standalone `chapter5/index.html`, same architecture as Ch2/Ch4 (fixed bg layer + scroll-content layer, IntersectionObserver beat reveals).

---

## Backgrounds

Two 2K images generated at `debug/`:

| File | Used for | Character |
|------|----------|-----------|
| `debug/bg_rainy_forest.png` | M1 background | Dark, heavy storm sky, navy tones, wet puddles |
| `debug/bg_foggy_forest.png` | M2 background | Muted, desaturated, half-dissolved in white mist |

Both share the same composition and tree silhouettes — same world, different weather. The transition reads as continuity, not a scene change.

**Final asset paths:** Move to `chapter5/backgrounds/` during build.

## Layer Stack

```
Layer 0 — .bg-image       position: fixed    z-index: 0    Forest background image
Layer 1 — .rain-layer     position: fixed    z-index: 1    CSS animated rain
Layer 2 — .fog-overlay    position: fixed    z-index: 2    Single element: blur overlay + breathing pulse animation (M2 only, hidden in M1)
Layer 3 — .scroll-content position: relative z-index: 3    All text beats
```

`.fog-overlay` doubles as the breathing pulse element — no separate `.breath-overlay`. It carries both `backdrop-filter: blur(2px)` and the `breathe` keyframe animation.

## Movement 1 — The Speedrun

### Background
- `bg_rainy_forest.png` fixed, `background-size: cover`, `background-position: center`
- CSS rain overlay: many thin cartoon raindrops, fast fall (~1.2s duration), dense

### Scroll density
Beats packed close — `rootMargin: '0px 0px -10px 0px'` (vs `-40px` in Ch4). Reveals fire almost immediately on scroll.

### Text style
- Reveal duration: **0.5s** (vs 0.85s in Ch4)
- `font-weight: 600`, tighter `line-height: 1.3`
- Some lines: enter with `skewX(1deg)` that snaps to `skewX(0)` — subtle instability
- Color: near-white on dark background (`#F0F0F0`)

### Script / Beat Structure

| ID | Text | Notes |
|----|------|-------|
| `beat1` | "Then…" | Single word, weight 400, italic, slow intro — deceptive calm |
| `beat2` | "we did something impressive." | Weight 600, 0.5s |
| `beat3` | "We speedran a relationship." | Weight 700, largest text in M1 |
| `beat4` | "Any% completion." | Monospace font, smaller — game HUD nod |
| `beat5` | "No tutorials." | Same monospace, stagger 150ms |
| `beat6` | "No save points." | Same, stagger 300ms |
| `beat7` | "I might have pressed" | Weight 400, standard |
| `beat8` | ""fast forward"" | Weight 700, slightly larger, quoted |
| `beat9` | "a bit too hard." | Weight 400, italic, the confession |

### Memory Flash (end of M1)
Triggered after `beat9` enters viewport. Scroll locks (`overflow: hidden` on `body`) for ~1s total (6×150ms images + 80ms glitch flash).

6 memory thumbnails flash fullscreen in sequence, each ~150ms:
1. Chocolate box (from Ch3)
2. Chat bubbles (from Ch4 Scene A)
3. Ramen bowl (from Ch4 Scene B)
4. 👉👈 arrows (from Ch4 Scene B)
5. "You laughed." burst (from Ch4 Scene C)
6. White blank flash

Then: single-frame color invert (CSS `filter: invert(1)`, 80ms) → hard cut to M2.

Memory thumbnail sources — resolve during implementation by checking actual image files in `chapter3/` and `chapter4/` asset folders. If no standalone image exists for a memory, create a CSS-drawn fullscreen recreation (dark bg + the key element centered) rather than borrowing a composite scene asset.

```javascript
function triggerMemoryFlash(onComplete) {
  const memories = [
    // Paths resolved during implementation from chapter3/ and chapter4/ asset folders
    { src: '../chapter3/backgrounds/...', label: 'chocolate' },
    { src: '../chapter4/backgrounds/phone_screen_glow.png', label: 'chat' },
    { src: '../chapter4/props/ramen_bowl.png', label: 'ramen' },
    // arrows, laugh burst — CSS overlays if no standalone image
  ];
  document.body.style.overflow = 'hidden';
  let i = 0;
  const overlay = document.querySelector('.memory-flash-overlay');
  overlay.style.display = 'block';

  const next = () => {
    if (i >= memories.length) {
      // glitch flash then complete
      document.body.classList.add('glitch-invert');
      setTimeout(() => {
        document.body.classList.remove('glitch-invert');
        overlay.style.display = 'none';
        document.body.style.overflow = '';
        onComplete();
      }, 80);
      return;
    }
    overlay.style.backgroundImage = `url(${memories[i].src})`;
    i++;
    setTimeout(next, 150);
  };
  next();
}
```

```css
.glitch-invert { filter: invert(1); }
```

## Hard Cut Transition

At the M1→M2 boundary:
- `bg-image` instantly swaps from `bg_rainy_forest.png` → `bg_foggy_forest.png` (no CSS transition — instant)
- Rain animation: `animation-duration` shifts from 1.2s → 2.5s, `opacity` drops from 0.7 → 0.35 (fewer, slower, drifting drops)
- `.fog-overlay` fades in over 0.8s
- Scroll unlocks

## Movement 2 — The Glitch

### Background
- `bg_foggy_forest.png` fixed
- Rain continues but slow: ~2.5s fall, low opacity, slight horizontal drift
- `.fog-overlay`: semi-transparent white layer (`rgba(255,255,255,0.12)`) + `backdrop-filter: blur(2px)`

### Breathing pulse
```css
@keyframes breathe {
  0%, 100% { opacity: 0.04; }
  50%       { opacity: 0.10; }
}
/* Applied to .fog-overlay (same element as the blur layer) */
.fog-overlay {
  position: fixed; inset: 0; z-index: 2; pointer-events: none;
  background: radial-gradient(ellipse at center, rgba(200,210,230,0.12) 0%, transparent 70%);
  backdrop-filter: blur(2px);
  animation: breathe 4s ease-in-out infinite;
  opacity: 0; /* hidden in M1, fades in on hard cut */
}
```

### Scroll density
Beats spread far apart — `rootMargin: '0px 0px -60px 0px'`, large `margin-bottom` between beats (~40vh). The page breathes.

### Text style
- Reveal duration: **1.1–1.4s**
- `font-weight: 400`, generous `line-height: 1.8`
- Blur fade-in: `filter: blur(4px) → blur(0)` on `.visible` (matches foggy bg)
- Color: soft off-white (`#E8ECF0`)

### Script / Beat Structure

| ID | Text | Notes |
|----|------|-------|
| `beat10` | "Somewhere in the middle…" | 1.1s, blur fade, weight 400 italic |
| `beat11` | "things got a little blurry." | 1.2s, blur fade |
| `beat12` | "Not wrong." | 1.0s, smaller, quieter |
| `beat13` | "Just…" | 0.9s, italic, the pause |
| `beat14` | "too much, too fast." | 1.3s, weight 500 |
| `beat15` | "You weren't pulling away." | 1.2s, no blur — clean reveal |
| `beat16` | "You were just trying to breathe." | **1.4s, weight 400, no blur, no skew — the emotional centre. Clean and clear.** |

### Chapter end
After `beat16`, a quiet fade-in of the next chapter button (~1.5s delay after beat16 fires). No fanfare.

## CSS Rain Animation

Cartoon-style raindrops as thin `<div>` elements generated by JS. Two states:

**M1 Rain (fast, dense):**
```css
@keyframes rainFall {
  from { transform: translateY(-10px) translateX(0); opacity: 0.7; }
  to   { transform: translateY(110vh) translateX(-20px); opacity: 0.4; }
}
```
- ~80 drops, 1.0–1.4s duration, random horizontal start, `width: 1.5px`, `height: 14px`, `border-radius: 2px`, `background: rgba(150,170,200,0.6)`

**M2 Rain (slow, drifting):**
- ~30 drops, 2.2–2.8s duration, more horizontal drift (`translateX(-35px)`), lower opacity (0.2–0.35)

Transition between states: JS updates each drop's `animation-duration` and opacity on the hard cut event.

## Verification Checklist

- [ ] M1 beats fire quickly — no perceptible delay between lines on normal scroll speed
- [ ] `skewX` glitch on select beats: visible but subtle (never more than 1–2deg)
- [ ] Memory flash: 6 images at ~150ms each + 80ms glitch = ~1s total scroll lock
- [ ] Hard cut: background swap is truly instant (no CSS transition on bg-image swap)
- [ ] Rain slows visibly after hard cut — user should notice the world calming
- [ ] M2 blur fade-in: text starts blurry and sharpens — not the reverse
- [ ] `beat16` ("You were just trying to breathe"): cleanest, slowest reveal on the page
- [ ] Breathing pulse: barely perceptible — 0.04→0.10 opacity only
- [ ] Mobile: rain drops don't cause layout overflow (use `overflow: hidden` on `.rain-layer`)
- [ ] Next chapter button fades in cleanly after beat16 — no abrupt appearance
